# MoA Internal Test Guide

## 목적
이번 내부 테스트의 목적은 MoA의 핵심 업무 흐름이 실제 사용 기준으로 자연스럽게 동작하는지 확인하는 것입니다.

특히 아래 3가지를 중점적으로 봐주세요.

- 로그인 및 승인 흐름
- 이슈 생성, 수정, 이동, 삭제
- Slack에서 Inbox로 들어오는 이슈 처리 흐름

## 접속 정보
- 서비스 URL: `https://moa-brown.vercel.app`
- 로그인 방식:
  - 이메일 회원가입
  - Google 로그인

## 우선 확인 흐름
### 1. 계정 및 승인
- 이메일 회원가입이 정상적으로 되는지
- Google 로그인으로 신규 진입이 되는지
- 승인 전에는 `Pending Approval` 화면이 보이는지
- 승인 후 다시 로그인하면 정상적으로 서비스에 들어오는지

### 2. 기본 이슈 관리
- 프로젝트 생성이 되는지
- 이슈 생성/수정/삭제가 되는지
- 상태 변경, 우선순위 변경, 담당자 지정이 되는지
- 이슈 상세 패널과 전체 상세 페이지가 정상적으로 열리는지
- 이슈 링크 복사가 되는지

### 3. Inbox 및 Slack 흐름
- Slack `/issue` 명령으로 Inbox에 이슈가 생성되는지
- Inbox에서 이슈를 다른 프로젝트로 이동할 수 있는지
- 이동 후에도 이슈가 정상적으로 열리고 수정되는지
- Slack 사용자 연결 시 reporter가 올바르게 매핑되는지

### 4. 일정 및 검색
- Dashboard의 `Today`, `This week`가 정상적으로 보이는지
- Calendar에서 일정이 없어도 화면이 정상적으로 보이는지
- Search에서 검색 및 필터가 정상적으로 동작하는지

### 5. 댓글 및 첨부
- 댓글 등록/삭제가 되는지
- 첨부 업로드/삭제가 되는지
- 이미지/파일 링크가 정상적으로 열리는지

## 성능 체감 체크
아래 항목은 기능 이상만큼 중요합니다.

- 메뉴 첫 진입 시 유독 느린 화면이 있는지
- 이슈 상세 패널 오픈 속도가 괜찮은지
- 같은 화면을 다시 열 때 속도가 나아지는지
- “멈춘 것처럼 느껴지는” 구간이 있는지

## 권한 및 접근
- 승인되지 않은 사용자가 주요 화면에 접근되지 않는지
- 일반 사용자가 관리자 기능에 접근되지 않는지
- 관리자 계정에서는 `Users` 메뉴와 승인 기능이 정상 동작하는지

## 제보 형식
이슈를 발견하면 아래 형식으로 전달해 주세요.

- 화면:
- 수행한 행동:
- 기대한 결과:
- 실제 결과:
- 재현 여부:
- 첨부:

예시:

- 화면: Inbox
- 수행한 행동: 테이블 뷰에서 이슈를 다른 프로젝트로 이동
- 기대한 결과: 프로젝트가 정상 변경되고 상세 패널이 유지됨
- 실제 결과: 에러 메시지 발생
- 재현 여부: 2회 재현
- 첨부: 캡처 또는 짧은 영상

## 참고
- 첫 요청은 다소 느릴 수 있습니다.
- 테스트 중 생성된 데이터는 운영 점검 후 정리할 예정입니다.
- 로그인/승인 이슈가 있으면 계정 이메일 주소와 함께 알려주세요.
